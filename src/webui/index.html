ReAct Agent
启用工具层（Agent 模式） 开启后新建对话将使用 ReAct 模式，Agent 会通过推理步骤调用工具；关闭则使用纯对话 Chat 模式。
L1 短期记忆（当前 session 推理步骤）
启用短期记忆 保留当前 session 最近 N 步的 Thought/Action/Observation 作为 scratchpad 注入 prompt。
启用步骤蒸馏 窗口溢出时用 LLM 压缩驱逐步骤，生成摘要注入当前 prompt，避免丢失推理上下文。
L2 中期记忆（近期对话历史）
启用近期历史 自动记录每轮对话的 Q&A，下次启动时加载最近 N 天的记录注入 system prompt。
启用滚动整合 超出最大条目数后，用 LLM 把最旧的一批条目蒸馏为摘要，每天最多自动整合一次。需 LLM 就绪。
L3 长期记忆（Qdrant 向量库）
启用长期记忆 跨 session 向量化存储全部历史 Q&A，每轮根据问题语义检索相关记忆注入 prompt。使用本地 Qdrant 向量库，需安装 qdrant-client 和 sentence-transformers。
启用知识蒸馏 每轮结束后调用 LLM 将 Q&A 提炼为一条精炼知识摘要再写入向量库，减少噪声、提升检索质量。关闭时仅存原始答案文本。
危险操作

清空所有记忆层（L1 短期 / L2 中期 / L3 长期），操作不可逆。

唤醒机制
Scheduler 引擎在后台独立线程中持续运行，每隔「唤醒间隔」秒检查一次是否有到期任务，到期则调用 Agent 执行。暂停引擎不会取消任务，只是跳过触发检查。
间隔越短响应越精准，但 CPU 占用略高
主动推送通知 开启后，push 模式的任务执行完毕时将通过绑定渠道向用户推送结果。关闭则全局静默（优先级高于单任务设置）。
